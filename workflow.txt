APPLICATION AND PLACEMENT TRACKER

Workflow 1 (Saman Waseem): Adding the 'Add New Application' feature. 

- The user would log in and be led to the dashboard.
- They would click the '+ Add New Application' button.
- A form would appear requiring: 
  a. Specification of type: Internship / Job
  b. Job Role / Title
  c. Company Name
  d. Job Description Link
  e. Salary Estimate
  f. Application Deadline
  g. Current Status: Pending / Applied / Interview Stage / Accepted / Rejected / Ghosted / Dropped
- The user then clicks 'save' and the form closes. They could also click 'cancel' to exit. In both cases, they would be led back to the main interface displaying their applications.
- The entry is added to their applications view.
- A confirmation dialogue box appear 'Application Added Successfully!'

Workflow 2 (S.M. Abdullah Shabbir) - Deleting an Application

- The user logs in and is taken to the dashboard.
- The user can see their applications list.
- The user locates the application they want to remove.
- The user clicks the "Delete" button (or trash icon) next to the application.
- A confirmation pop-up appears asking:
  “Are you sure you want to delete this application?”
  Options: Yes / No
- If the user clicks "Yes":
  - The application is removed from the list.
  - The user remains on the dashboard.
  - A confirmation message appears: “Application Deleted Successfully!”
- If the user clicks "No":
  - The confirmation pop-up closes.
  - No changes are made to the application list.




Workflow 3 (Bareerah Shahid)

Editing an Existing Application
The user logs in and is taken to the dashboard.
They see a list/table of all saved applications.
The user clicks on an existing application entry.
They click an “Edit” button (or pencil icon).
A form opens with the already-filled details:
a. Internship / Job
b. Job Role / Title
c. Company Name
d. Job Description Link
e. Salary Estimate
f. Application Deadline
g. Current Status
The user updates one or more fields.
The user clicks “Save Changes”.
The form closes and the user is taken back to the dashboard.
The updated information appears in the applications view.
A confirmation dialogue appears: “Application Updated Successfully!”
If the user clicks “Cancel”, no changes are saved and they return to the dashboard.
